## Context
SeyalRun v2 is a microservices PAM (Privileged Access Management) console embeddable in
Zabbix. Security is the product, not a feature. A pre-merge audit found 3 critical issues
plus README claims that the code contradicts, and **zero automated tests**. This is the
cheapest moment to fix it: nothing is committed, no users, no migrations to honor.

## Vision

### 10x Check
Today's frame is "JumpServer parity." The 10x frame is **provable security**: a CISO
chooses this because it *demonstrates* its guarantees continuously rather than asserting
them. Credentials injected by the broker and never shown to a human; every keystroke in a
hash-chained audit log no admin can quietly edit; a green/red attestation badge backed by
the same invariant tests that run in CI.

### Platonic Ideal
Every privileged action is identity-bound, policy-gated, recorded, tamper-evident, and
observable in real time — and the guarantees are continuously self-verified, not asserted.
Breach containment by design: short-lived everything, zero standing credentials, least
privilege default.

## Baseline: Approach C (Security baseline as the MVP)
- H1: metrics-service dashboard — wire `Depends(_require_service_token)` (currently unauth).
- H2: fail-fast on missing secrets — shared `libs/secrets` required-field validator used by
  all 9 services (no more empty-string defaults that silently disable security).
- H3: webhook HMAC mandatory — refuse to boot / reject if `ZABBIX_WEBHOOK_HMAC_SECRET` unset.
- M1: scope CSP `frame-ancestors` to the Zabbix origin (not `*`).
- M2: SSH host-key verification (replace `known_hosts=None` with TOFU/managed known_hosts).
- M3: bind credential-reveal token to credential_id + subject.
- README reconciled to reality.
- pytest suite (CI-runnable) proving: service-token missing/wrong-audience rejected;
  gateway blocks `internal/*` + RBAC matrix; webhook HMAC reject + no-secret refuses boot;
  AES-GCM encrypt/decrypt round-trip.

## Contested decisions resolved (CEO review 2026-06-28)
- **Param safety (T11):** allowlist job params per job-template; automation rejects non-declared keys before executors. HMAC authenticates the sender, not the content.
- **Fail-fast rollout (H2):** hard-fail immediately (greenfield, no running env to break).
- **Metrics error handling:** per-panel `{degraded:true}` flag + scoped SQLAlchemy catch — never fake zeros on a security dashboard.
- **Metrics coupling:** each service exposes read-only `v_metrics_*` views as a schema contract; metrics-service queries the views, plus a schema-drift CI test.
